                    SmartDoc QA — 智能文档问答系统
                         完整功能介绍

一、项目简介
--------------------------------------------------------------------------------
SmartDoc QA 是一款基于 RAG（检索增强生成）技术的智能文档问答系统。用户上传
PDF 或 TXT 文档后，系统自动提取文本、分块并建立向量索引。用户以自然语言提问，
系统会检索最相关的文档片段，结合 DeepSeek / Kimi 大语言模型生成基于原文的精准
回答，并自动标注引用来源。

架构：前后端分离（Spring Boot 3.2 + Vue 3 + Element Plus）
端口：前端 http://localhost:5173  /  后端 http://localhost:8080


二、核心工作流程
--------------------------------------------------------------------------------
  [上传文档] → [PDF/TXT文本提取] → [智能分块(500字/块,50字重叠)]
       ↓
  [TF-IDF n-gram 向量化] → [Lucene HNSW 向量索引]
       ↓
  [用户提问] → [向量检索(最相关的Top-3块)] → [LLM生成回答 + [1][2][3]引用标注]
       ↓
  [流式输出到前端→ 逐字显示] → [追问建议生成] → [对话历史持久化]


三、功能总览（共66项）
--------------------------------------------------------------------------------

  【用户系统】(6项)
  1. 用户注册 — 校验用户名唯一性、密码长度
  2. 用户登录 — Session 机制保持登录状态
  3. 用户登出 — Session 销毁 + 前端清除
  4. 路由鉴权 — 未登录自动跳转登录页
  5. 用户首字头像 — 导航栏显示
  6. 完整登录注册 UI — 左右双栏品牌展示

  【文档管理】(12项)
  7.  上传 PDF/TXT 文件 — 格式自动校验
  8.  多文件上传 — 一次选择多个文件，Promise.allSettled 并发上传
  9.  上传进度条 — 实时显示已完成/总数百分比
  10. PDF 文本提取 — Apache PDFBox 引擎
  11. TXT 文本提取 — UTF-8 编码直读
  12. 文本智能分块 — 500 字符/块，50 字符重叠
  13. 文档列表 — 按用户归属、按上传时间倒序
  14. 按状态筛选 — 全部 / 已完成(绿) / 处理中(黄) / 失败(红)
  15. 按文件名搜索 — 实时模糊匹配
  16. 文档重命名 — 双击标题行内编辑
  17. 删除文档 — 级联删除：对话 → 消息 → Lucene 索引 → 物理文件
  18. 重新处理 — 失败文档重试按钮，清除旧索引重新提取

  【向量化与检索】(7项)
  19. 本地 TF-IDF + n-gram 混合向量 — 512 维，无需外部 API
  20. 全语料 IDF 批量计算 — 提高检索精度
  21. L2 归一化 — 余弦相似度标准
  22. Apache Lucene 9 HNSW 索引 — 高性能近似最近邻检索
  23. 余弦相似度排序 — 返回最相似 Top-K 块
  24. Chroma 向量数据库 — 预留接口
  25. 智谱/DeepSeek Embedding API — 预留支持

  【智能问答 RAG】(12项)
  26. 向量检索 — 问题向量化后比对文档块，取 Top-3
  27. 多轮对话记忆 — 最近 6 条消息作为上下文
  28. LLM 双模型 — DeepSeek Chat / Kimi (Moonshot) 自由切换
  29. System Prompt 优化 — 要求 LLM 使用 [N] 标注引用来源
  30. 流式 SSE 输出 — 逐 token 推送，打字机效果
  31. 可随时停止生成 — AbortController 中断 SSE 连接
  32. 引用溯源标注 — 回答中 [1][2][3] 自动渲染为可点击蓝色徽章
  33. 引用原文可展开 — el-collapse 折叠面板，含完整原文
  34. 引用点击联动 — 点击引用编号自动展开面板并黄色高亮对应条目
  35. 消息持久化 — 用户消息 + AI 回答 + 引用 JSON 存入数据库
  36. 重新生成 — 保留原问题，重新调用 LLM 生成新回答
  37. Markdown 渲染 — 支持标题/代码块/表格/列表/粗体/斜体/引用

  【对话管理】(9项)
  38. 创建对话 — 每个文档可有多轮独立对话
  39. 对话列表 — 按文档过滤 / 查看全部
  40. 对话切换 — 同一文档的多对话间自由切换
  41. 对话重命名 — 双击标题行内编辑
  42. 首条消息自动标题 — 截取问题前 30 字
  43. 删除对话 — 单条 + 批量勾选删除
  44. 清空对话 — 保留对话容器，清除全部消息
  45. 导出对话 — 导出为 Markdown 文件 (.md) 下载
  46. 对话侧栏 — 聊天页右侧边栏显示当前文档所有对话

  【交互增强】(7项)
  47. 追问建议 — AI 回答后自动生成 2-3 条追问，点击直接发送
  48. 消息复制 — 点击复制按钮将回答复制到剪贴板
  49. 智能滚动 — 流式输出时自动滚到底，用户翻看历史时暂停自动滚动
  50. 回到底部按钮 — 向上翻看后出现浮动按钮，一键回底
  51. 快捷键 — Ctrl+K 新建对话 / Ctrl+J 切换对话 / Shift+Enter 换行
  52. 消息淡入动画 — 新消息平滑出现
  53. 流式闪烁光标 — 回答生成中的动画指示

  【仪表盘】(10项)
  54. 4 列统计卡片 — 文档总数 / 对话次数 / 消息数 / 存储用量
  55. 今日增量徽章 — 每个卡片显示 +N 今日
  56. 文档状态进度条 — 已完成(绿)/处理中(黄)/失败(红) 可视化
  57. 最近活动时间线 — 上传 / 对话活动带时间戳
  58. 最近文档 Top 5 — 快速跳转
  59. 热门文档 Top 3 — 按对话次数排名
  60. 三步引导 — 上传 → 处理 → 提问 的图文流程
  61. 核心能力介绍 — RAG / 原文溯源 / 多轮记忆 / 多格式解析
  62. 空状态优化 — 新用户大 Hero 卡片 + 首份文档上传引导
  63. 快捷操作区 — 上传 / 打开最近文档 / 刷新

  【系统配置】(3项)
  64. LLM 提供商选择 — DeepSeek / Kimi 下拉切换
  65. Embedding 模型选择 — 本地 TF-IDF(可用) / 智谱 / DeepSeek(预留)
  66. 可调参数 — 分块大小(100-2000) / 重叠大小(0-500) / Top-K(1-10)

七、系统优点
================================================================================

  1. 【真正的 RAG 架构】
     并非简单的"上传文件 → 全文搜索 → 让AI读"，而是完整实现了：
     文本提取 → 智能分块 → 向量化嵌入 → 近似最近邻检索 → LLM生成。
     每一个环节都是独立模块，可替换、可升级。

  2. 【零外部依赖的本地向量化】
     Embedding 默认使用自研 TF-IDF + n-gram 混合向量方案，计算完全在本地
     完成，不依赖任何外部 Embedding API（如 OpenAI Embeddings），无需联网，
     无 API 费用，非常适合演示和低预算场景。

  3. 【双 LLM 模型自由切换】
     同时支持 DeepSeek Chat 和 Kimi（月之暗面）两款大模型。用户在对话界面
     即可通过下拉菜单实时切换，追问建议和重新生成都自动跟随当前选中模型。

  4. 【流式 SSE 输出 + 可中断】
     基于 SSE（Server-Sent Events）实现逐 token 流式推送，前端用原生
     ReadableStream 读取，打字机效果流畅自然。支持通过 AbortController
     随时中断生成，保留已生成的内容。

  5. 【引用溯源 + 原文可查】
     LLM 回答中的 [1] [2] [3] 不是装饰 —— 每个编号对应检索到的真实文档
     片段。用户点击编号即可展开引用原文面板、看到完整的原文内容。答案
     有据可查，杜绝"幻觉"问题。

  6. 【多轮对话记忆】
     每次提问携带最近 6 条对话历史作为上下文，AI 可以理解"它"、"那个"等
     指代。聊得越多，回答越精准。

  7. 【Markdown 渲染 + 代码块】
     AI 回答中的 Markdown 格式（标题、粗体、表格、代码块、列表等）被正确
     渲染为富文本。代码块有深色背景和高亮，阅读体验接近 IDE。

  8. 【追问建议智能生成】
     每次 AI 回答后，自动调用 LLM 基于对话上下文生成 2-3 条追问建议。
     用户点击即可继续深入探讨，降低了提问门槛。

  9. 【多文件并行上传】
     支持 Ctrl/Shift 多选 PDF/TXT 文件，使用 Promise.allSettled 并发上传，
     上传后异步处理，不阻塞用户操作。

  10.【完善的仪表盘】
      首页提供文档总数、对话数、今日增量、存储用量等关键指标，文档状态
      分布可视化（绿色已完成/黄色处理中/红色失败），最近活动时间线和热门
      文档排名让用户一目了然。

  11.【对话管理功能丰富】
      支持创建多轮对话、对话间切换、重命名、导出为 Markdown、批量删除、
      清空消息（保留容器）、首条消息自动标题。所有操作都有二次确认。

  12.【美观的 UI 设计】
      统一的蓝色主题（#2563eb）、一致的圆角/阴影/间距设计语言、文件图标
      区分 PDF/TXT、状态徽章颜色编码、移动端友好的响应式布局。字体大小
      经过精心调整（整体上调 1-2px），阅读舒适。

  13.【完整的用户系统】
      注册/登录/登出 + Session 鉴权 + 路由守卫。所有数据按用户隔离。

  14.【键盘快捷键】
      Ctrl+K 新建对话、Ctrl+J 切换对话、Enter 发送、Shift+Enter 换行。
      熟练用户可脱离鼠标操作。

  15.【可扩展的配置设计】
      application.yml 中所有关键参数可配：LLM 提供商、API Key、模型名、
      分块大小、重叠量、Top-K 等。Embedding 模块预留了智谱和 DeepSeek
      的 API 接口，向量存储预留了 Chroma 支持。

  16.【数据库内置，零配置启动】
      使用 H2 文件数据库，无需安装 MySQL/PostgreSQL。首次启动自动建表。
      适合演示、教学和快速部署。

  17.【智能轮询机制】
      文档上传后前端自动轮询后端状态，直到处理完成才停止。失败文档有
      重新处理按钮，不会因为一次失败而废弃。

  18.【完整的技术栈覆盖】
      系统涵盖了现代 Web 全栈开发的核心技术领域：RESTful API 设计、
      异步任务处理(@Async)、文件上传处理、向量数据库(Lucene)、NLP 文本
      处理(PDFBox)、流式传输(SSE)、前端状态管理、组件化开发(Vue 3 SFC)、
      路由设计等。
